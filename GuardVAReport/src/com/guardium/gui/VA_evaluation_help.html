IBM InfoSphere Guardium Vulnerability Assessment Evaluation Edition

With Guardium Vulnerability Assessment (VA) Evaluation, you can scan your data repositories using a set of Guardium predefined query-based vulnerability tests and CVE vulnerabilities, and assess the overall security posture of your databases.

Overview

What is Guardium Vulnerability Assessment?

The Guardium Vulnerability Assessment solution is the first step in the security and compliance lifecycle management for any IT environment. Tests, predefined and custom, along with a process workflow, allow organizations to identify and address database vulnerabilities in an automated fashion/proactively improving configurations and hardening infrastructures.

Note: Guardium Vulnerability Assessment requires access to the databases it evaluates.

How does Guardium Vulnerability Assessment Work?

The Guardium Vulnerability Assessment application enables organizations to identify and address database vulnerabilities in a consistent and automated fashion. Guardium's assessment process evaluates the health of your database environment and recommends improvement by:

·       Assessing system configuration against best practices and finding vulnerabilities or potential threats to database resources, including configuration and behavioral risks. For example, identifying all default accounts that haven't been disabled; checking public privileges and authentication methods chosen, etc.

·       Finding any inherent vulnerabilities present in the IT environment, like missing security patches.

·       Recommending and prioritizing an action plan based on discovered areas of most critical risks and vulnerabilities. The generation of reports and recommendations provide guidelines on how to meet compliance changes and elevate security of the evaluated database environment.

Step procedure for VA Evaluation

1.     Install product - After downloading the install file, click on the VA evaluation.exe file. The product will self install.

 

2.     Accept license - When you run the program for the first time, you will need to accept a 30-day license. View the license terms in one of 18 languages, the default is English.

 

3.     Obtain privileges - Use your admin privilege to connect to the databases or create minimum privilege to connect to the databases. Minimum privilege is Guardium-only privilege. Nothing on the database is modified. The download page will contain a file named "Privilege_require_to_run_VA.zip". This file contains the .SQL files needed to create minimum privilege. The file list is:

·       gdmmonitor-db2.sql

·       gdmmonitor-ifx.sql

·       gdmmonitor-mss.sql

·       gdmmonitor-mss-SA.sql

·       gdmmonitor-netezza.sql

·       gdmmonitor-ora.sql

·       gdmmonitor-ora-container.sql

·       gdmmonitor-syb.sql

·       gdmmonitor-sybaseIQ.sql

·       gdmmonitor-teradata.sql

·       Jconnect_SybaseIQ_requirement.txt

 

4.     Connect to your target database - There are eight databases supported - Oracle, MS-SQL Server, DB2, Informix, Netezza, Sybase, Sybase IQ, Teradata. Go to the Test Configuration menu to create a new datasource connection. The Test Configuration menu choices are: Edit, Delete, New, Run.

Click New to go to the datasource configuration menu.

A datasource identifies a specific database instance or file on a remote system that stores data. Datasources can be shared, but access is restricted according to the roles assigned to both the datasource and the application that uses it.

After filling in the login/password, hostname/IP, port information, click Test Connection to confirm that the datasource can be successfully connected to. Then click Apply.

 

5.     Go to Test Configuration menu to run predefined query-based tests and CVE tests. Select the database to assess, and then click Run Vulnerability Assessment. Once you run VA, a prompt will appear asking for the password, if it is not cached in memory. If the password is cached in memory, meaning you executed this once or more times already, the password prompt will not appear. At the popup, enter the test results name or use the default test results name. Progress is shown by a moving image. Click the STOP Sign to stop the process.

 

6.     The program automatically displays an HTML-based report in your browser. Two types of tests are shown:

  • Query-based tests - These are general hardening tests associated with the chosen database.
  • Common Vulnerability Exposure (CVE) tests - These are known industry-standard tests identified by CVE's common identifier and associated with the chosen database.

For each database assessment, the report will show:

  • an overall test grade, displayed as a percentage of all completed tests passed.
  • a listing of tests that passed, a listing of tests that failed, and other tests. (The "other" category are tests that did not complete for a variety of reasons or do not apply to the particular database.)
  • a detailed table with Result/Test name/Description of Test/Recommendation.

The result column also shows a test Category (privilege, configuration, authentication, version, and other) and displays Severity (critical, major, minor, informational, caution, other).

In each database type, there will be ten tests which provide complete information similar to that contained in the full VA product. For the remaining test results, there will be more limited information.
